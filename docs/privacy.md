# QuotaGlance 隐私说明

> 文档状态：1.0 设计草案  
> 更新日期：2026-07-14
> 当前阶段：0.1.5 跨平台社区预览，尚无正式稳定版
> 联系方式：maorongkang@gmail.com

## 先说明当前状态

QuotaGlance 目前提供 `0.1.5` 源码和未签名/未公证跨平台社区预览软件，但没有正式稳定版、商店签名包或由项目运营的 QuotaGlance 服务。预览版可在本机处理模拟数据；连接用户已经安装并登录的 Codex CLI、统一版 ChatGPT 应用或旧版 Codex 应用运行组件时，也会在内存中处理账号模式和额度数据。项目不会因此把数据上传给 QuotaGlance 开发者。

当前代码已采用只读类型化协议、字段白名单和最小 WebView Capability，额度快照仅保存在内存；窗口模式、置顶、穿透、主题和登录时启动偏好保存在应用配置目录，更新器和 production sidecar 尚未完成。本文仍以 1.0 目标隐私边界为准，正式稳定版发布前必须用实际代码和安装包重新核对。

## 一句话说明

QuotaGlance 是一款本地额度查看工具。它不会建设 QuotaGlance 账号系统或中转服务器，不会把您的 Codex Token、提示词、代码和额度历史发送给 QuotaGlance 开发者。

正常情况下，QuotaGlance 只向本机的 Codex App Server 读取当前账号模式和额度状态。登录凭据仍由 Codex 管理。

## 我们会处理哪些数据

### 当前额度信息

为了显示额度，应用会处理 Codex App Server 返回的必要字段，例如：

- 当前认证模式，例如 ChatGPT 登录或 API Key 模式；
- 套餐类型（服务端提供时）；
- 额度桶名称、已用或剩余比例；
- 额度窗口时长和重置时间；
- 服务端提供的 credits 和可用重置次数；
- 数据状态、最后成功更新时间和下一次重试时间。

这些数据用于当前界面展示和故障恢复。1.0 默认只保存在内存中，退出应用后不会形成额度历史。

Codex App Server 可能返回更多账号字段。QuotaGlance 会在 Rust 层只保留展示所需字段，邮箱、账号 ID、workspace ID 等非必要信息不会发送给前端，也不会写入设置或日志。

### 本地偏好

为了让应用下次启动时保持您的选择，`0.1.5` 已在设备本地保存窗口模式、置顶、鼠标穿透、主题和登录时启动状态；语言、提醒、更新和完整窗口位置仍属于后续设置闭环。偏好 Schema 覆盖以下数据：

- 浮球或卡片的位置与模式；
- 置顶、鼠标穿透和锁定状态；
- 语言、主题和提醒阈值；
- 是否开机启动；
- 更新通道和最后一次检查更新时间。

这些偏好不需要上传到 QuotaGlance 服务器，因为项目不计划提供这类服务器。

### 更新检查所需信息

检查更新时，发布站点通常需要知道：

- 当前应用版本；
- 操作系统与 CPU 架构；
- 请求更新时间。

和普通网页下载一样，托管服务还可能在网络日志中看到 IP 地址、User-Agent 和请求时间。实际更新托管方、链接及其隐私规则会在正式发布前写入用户手册。更新请求不会附带 Codex Token、账号信息或额度值。

## 我们不会读取或收集什么

正常使用 QuotaGlance 时，不读取或收集：

- Codex access token、API Key、刷新令牌、Cookie 或密码；
- `auth.json` 内容；
- 操作系统凭据库中的 Codex 凭据；
- 邮箱、账号 ID、workspace ID 和联系人；
- 提示词、聊天历史、Codex 任务内容；
- 源代码、项目文件、Git 仓库内容和终端历史；
- 浏览器历史、书签或网页内容；
- 麦克风、摄像头、位置、屏幕画面和通讯录；
- 广告标识、跨应用跟踪信息或行为画像；
- 默认的额度历史、使用趋势或个人效率统计。

QuotaGlance 不出售个人数据，不投放个性化广告，也不使用第三方分析脚本。

## Codex 登录凭据如何处理

Codex 官方支持三种凭据存储方式：

- `file`：在 `CODEX_HOME` 下的 `auth.json` 保存凭据，该文件包含 access token；
- `keyring`：保存在操作系统凭据库；
- `auto`：优先使用凭据库，不可用时回退到 `auth.json`。

在 QuotaGlance 的正常路径中：

1. QuotaGlance 启动本机受控的 Codex App Server；
2. App Server 根据 Codex 配置读取并使用登录状态；
3. App Server 向 Codex 官方服务查询额度；
4. QuotaGlance 只接收并展示经过筛选的额度数据。

也就是说，QuotaGlance 本身不读取 `auth.json`、不访问系统 keyring、不构造 `Authorization` Header，也不把 Token 传给 React 界面。

QuotaGlance 不会替您登录、登出、购买额度、发送额度提醒邮件或兑换重置次数。

## 数据会发送到哪里

### 正常额度查询

```text
QuotaGlance 界面
  ↕ 本机 Tauri IPC
QuotaGlance Rust Core
  ↕ 本机 stdio JSONL
Codex App Server
  ↕ 由 App Server 建立的官方服务连接
Codex 官方服务
```

QuotaGlance WebView 不直接请求 ChatGPT 或 Codex 接口。Rust Core 与 App Server 通过本机子进程标准输入/输出通信，不监听本地网络端口。

App Server 与 OpenAI 之间的数据处理受您的登录方式、账号或工作区设置以及 OpenAI 的相关条款约束。QuotaGlance 无法替 OpenAI 决定其保留和处理规则。Codex 官方认证文档说明，ChatGPT 登录与 API Key 登录适用的管理和数据处理设置可能不同。

### 软件更新

应用只在检查或下载更新时访问预先配置的更新地址。更新包必须通过 Tauri 签名验证，并保留 Windows/macOS 的系统代码签名。更新服务与额度查询相互独立。

### 用户主动打开的链接

关于页、帮助页或更新说明中的外部链接，只在您点击后交给系统默认浏览器打开。目标网站可能按自己的隐私政策处理常规网络数据。

## 不提供凭据读取兼容模式

QuotaGlance 不实现也不保留读取 `auth.json`、访问系统 keyring、解析 access token/JWT 或直接调用非公开 `wham` HTTP 接口的备用 Provider。App Server 不可用时，应用显示明确错误并保留可用的最后成功快照，不会要求用户交出凭据，也不会自动切换到私有接口。重置机会只用于展示数量和到期时间，应用不提供购买、兑换或消耗入口。

## 数据保存在哪里、保存多久

| 数据 | 保存位置 | 保存时间 |
|---|---|---|
| 当前额度快照 | 应用内存 | 运行期间；退出后清除 |
| 最后成功额度 | 应用内存 | 运行期间，用于刷新失败时避免显示假零值 |
| 窗口、主题、提醒等偏好 | 当前用户的本地应用数据目录 | 直到您重置设置或手动删除 |
| 应用运行日志 | 当前用户本地；生产默认最小化 | 保留策略在实现后确定，不含敏感字段 |
| 崩溃报告与遥测 | 默认不启用 | 不适用 |
| 额度历史 | 1.0 默认不保存 | 不适用 |

项目不使用 MySQL、Redis 或云数据库。未来如果增加本地历史功能，必须由用户主动开启，明确保留期限和删除入口，并在启用前更新本文。

操作系统、安装器或安全软件可能生成自己的安装日志、崩溃记录和事件日志，这些数据由相应系统组件管理，不会自动发送给 QuotaGlance 开发者。

## 日志会记录什么

为排查问题，应用可以记录：

- 应用和 App Server 的非敏感版本；
- 启动、刷新、重试和状态变化；
- 脱敏错误码；
- 耗时、重试次数和缓存状态。

日志不得记录：

- Token、API Key、Cookie 或认证 Header；
- `auth.json` 和原始 JSONL 内容；
- 邮箱、账号 ID、workspace ID；
- 提示词、代码、项目名和文件内容；
- 完整用户名、主目录、`CODEX_HOME` 或外部 CLI 路径；
- 原始额度响应和 reset credit 的 opaque ID。

默认不启用遥测或自动崩溃上报。如果未来引入，必须先提供清晰说明、数据预览和主动选择，不得通过版本升级默认打开。

## 应用需要哪些权限

计划中的 1.0 权限如下：

| 权限或能力 | 默认行为 | 用途 |
|---|---|---|
| 网络 | App Server 查询官方服务；应用可检查更新 | 显示额度和获取签名更新 |
| 本地应用数据目录 | 使用 | 保存最小偏好 |
| 启动本机 sidecar | 使用 | 运行 Codex App Server |
| 开机启动 | 默认关闭，由用户选择 | 登录系统后显示浮球 |
| 系统通知 | 按需申请，由用户选择 | 本地额度提醒 |
| 系统凭据库 | QuotaGlance 不访问 | 登录状态由 App Server 自行处理 |
| 麦克风、摄像头、位置、通讯录、屏幕录制 | 不申请 | 与产品功能无关 |

React WebView 不获得通用文件系统、Shell 或任意网络访问权限。

## 如何删除本地数据

`0.1.5` 已把窗口模式、主题、置顶、穿透、登录时启动和提醒配置保存到应用配置目录；额度快照仍只保存在内存，并随程序退出清除。删除本地数据时可采用以下方法：

1. 在设置中重置 QuotaGlance 偏好；
2. 完全退出应用后，删除用户手册列出的 QuotaGlance 应用数据目录；
3. 卸载应用，并按需删除卸载器保留的偏好文件。

删除 QuotaGlance 数据不会删除 Codex 登录状态、`auth.json`、系统凭据库内容或 Codex 自身数据。QuotaGlance 也不会在卸载时擅自修改这些内容。

如果用户主动导出过诊断文件，还需要删除其选择的导出副本。正式版默认不自动上传诊断文件。

## 您可以做出的选择

- 不开启开机启动；
- 不授予系统通知权限；
- 手动检查更新，而不是使用计划中的每日检查；
- 随时退出应用，清除内存中的额度快照；
- 重置或删除本地偏好；
- 不提交任何诊断材料。

核心额度查看不应以同意遥测、广告跟踪或云同步为前提。

## 儿童数据、出售与共享

QuotaGlance 不面向儿童建立账号或画像，不主动识别或收集年龄。项目不出售个人数据，不与广告商或数据经纪商共享数据。

正常额度查询涉及 Codex App Server 和 Codex 官方服务；软件更新涉及最终选定的发布托管服务。除此之外，1.0 不计划接入第三方分析、广告或崩溃收集服务。

## 本说明的变更

新增以下能力前必须更新本说明并重新征得适当选择：

- 遥测或自动崩溃上报；
- 额度历史和趋势；

- 将数据用于当前额度展示之外的目的。
