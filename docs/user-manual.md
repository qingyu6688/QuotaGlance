# QuotaGlance 用户手册

> 手册版本：0.1.6 跨平台预览版 / 1.0 规划版
> 对应产品版本：0.1.6（公开预览），1.0.0（目标）
> 更新日期：2026-07-14
> 维护邮箱：maorongkang@gmail.com

## 1. 使用前说明

QuotaGlance（额度一览）是一款面向 Windows、macOS 与 Linux 的本地 Codex 额度助手，通过浮球、展开卡片和系统托盘展示当前额度和重置时间。

**当前提供 `0.1.6` GitHub 跨平台社区预览，但没有正式稳定版。** Windows 安装包未进行 Authenticode 签名，macOS DMG 仅使用 ad-hoc 签名且未公证，Linux 尚未覆盖全部发行版与桌面环境；所有平台产物都不捆绑 Codex App Server sidecar。请只从 [GitHub Releases](https://github.com/qingyu6688/QuotaGlance/releases) 下载，并先阅读对应版本说明。

QuotaGlance 是独立第三方工具，与 OpenAI 无隶属、授权或背书关系。

## 2. 适用范围

### 2.1 平台范围

| 平台 | 当前预览与 1.0 计划 |
|---|---|
| Windows 11 x64 | 1.0 目标；当前预览待完整实机验收 |
| Windows 10 22H2 | 仅对仍获 ESU 安全更新的设备尽力支持 |
| macOS 13 及以上，Apple Silicon | 提供预览构建；尚无实机、Developer ID 与公证证据 |
| macOS 13 及以上，Intel | 提供预览构建；尚无实机、Developer ID 与公证证据 |
| Linux x64、ARM64 | 提供社区预览；1.0 支持范围待实机反馈确定 |
| Windows ARM64 | 计划在 1.1.x 评估 |

### 2.2 使用前提

正式版预计需要：

1. 设备能够正常访问 Codex 服务。
2. 用户已通过包含 Codex 的 ChatGPT 桌面应用、旧版 Codex 桌面应用或 Codex CLI 完成登录。
3. 如使用 ChatGPT 订阅额度视图，当前认证方式需要由 Codex 服务提供对应的额度信息。
4. 企业设备上的代理、安全软件和执行策略允许 QuotaGlance 启动本机已经安装的 Codex App Server。

QuotaGlance 1.0 不计划提供自己的登录页，也不会要求用户把 Token、API Key 或 auth.json 内容粘贴到应用中。

## 3. 预览版安装与开发运行

`0.1.6` 安装资产发布在 [GitHub Releases](https://github.com/qingyu6688/QuotaGlance/releases/tag/v0.1.6)。请选择与系统和 CPU 架构匹配的文件，并使用同一 Release 中的 `SHA256SUMS.txt` 核对完整性。预览版未完成商业签名与公证，也没有 bundled sidecar，不应在要求正式签名与受控供应链的生产环境中部署。

开发者仍可在项目根目录执行 `npm ci` 后使用 `npm run dev` 查看模拟数据前端预览；具备 Rust、Tauri 和系统依赖时，可使用 `npm run tauri -- dev` 启动桌面调试程序。

### 3.1 Windows

1. 从 GitHub Release 下载 x64 NSIS `.exe` 或 MSI `.msi`。
2. 核对版本、文件名和 SHA-256；当前预览版显示“未知发布者”属于已知限制。
3. 完成安装并启动 QuotaGlance。
4. 首次出现 Windows 安全提示时，先核对发布者、版本和下载来源；不要使用来源不明的未签名包。

### 3.2 macOS

1. 下载与设备架构匹配的 Apple Silicon 或 Intel DMG。
2. 将 QuotaGlance 拖入“应用程序”。
3. 从“应用程序”启动；当前预览版仅 ad-hoc 签名且未公证，Gatekeeper 可能阻止运行。
4. 如果系统提示应用已损坏或无法验证，请先确认下载来源和 SHA-256，不要通过全局关闭 Gatekeeper 绕过。

当前 macOS 版会优先检查统一版 `ChatGPT.app`，随后兼容旧版 `Codex.app`，最后回退到 `PATH` 和常见 CLI 目录。系统级与用户级固定资源位置均受支持：

- `/Applications/ChatGPT.app/Contents/Resources/codex`
- `~/Applications/ChatGPT.app/Contents/Resources/codex`
- `/Applications/Codex.app/Contents/Resources/codex`
- `~/Applications/Codex.app/Contents/Resources/codex`

只接受完整应用包内的普通可执行文件，不扫描其他应用、缓存或临时目录，也不跟随把路径导向应用目录之外的符号链接。这些文件仍属于用户本机的外部安装，当前版本尚未校验 bundle identifier 或 OpenAI 代码签名身份。自动发现已经由代码和 macOS 条件测试覆盖，但尚未在 Intel/Apple Silicon 真机完成运行验收。

### 3.3 Linux

1. x64 或 ARM64 用户下载对应的 `.AppImage` 或 `.deb`。
2. 使用 `SHA256SUMS.txt` 核对文件；AppImage 需要具备可执行权限。
3. 发行版需要提供 WebKitGTK 等 Tauri 运行依赖；当前尚未覆盖所有桌面环境。

### 3.4 首次读取

首次启动后，QuotaGlance 会尝试：

1. 发现用户已经安装的 Codex CLI、Windows 受管运行时、macOS 统一版 ChatGPT 应用或旧版 Codex 应用，并启动只读 App Server；
2. 完成本地协议初始化。
3. 读取当前认证状态。
4. 在适用的 ChatGPT 认证模式下读取额度。
5. 显示额度，或在 15 秒内给出明确的错误状态。

正常路径中，QuotaGlance 不读取 `auth.json`，不访问系统凭据库，也不接收或保存 Token；应用不会直接调用非公开 `wham` HTTP 接口。认证与服务请求由 Codex App Server 处理。

## 4. 认识额度信息

### 4.1 剩余比例

Codex App Server 返回窗口内的已用比例，QuotaGlance 计划在本地 Rust 层统一换算为剩余比例。例如，服务端返回已用 25% 时，界面显示剩余 75%。

界面不会把未知值、解析失败或未登录状态显示成 0%。因此：

- “剩余 0%”只应在数据源明确表明额度用尽时出现。
- “未知”表示当前无法可靠确定，不等同于额度用尽。
- “旧数据”表示该数值曾经成功读取，但最近一次刷新失败。

### 4.2 额度桶与窗口

一个账号可能返回一个或多个额度桶，每个桶可能包含 primary、secondary 等窗口。QuotaGlance 在数据层兼容这些动态窗口：悬浮球使用服务端标记为 `weekly` 的周额度，展开卡片同时显示服务端实际返回的短周期/5 小时额度。

QuotaGlance 计划：

- 从用户选定的额度桶中优先选择周额度；该桶没有周额度时回退到其他桶的首个周额度。
- 短周期优先选择服务端标记的短周期窗口，再兼容不是周周期的 primary 窗口。
- 保留原始额度桶标识和可选名称。
- 短周期仅在服务端实际返回时出现；月度和未知窗口继续安全兼容，不被猜测为 5 小时额度。
- 周额度缺失时浮球显示“暂无周额度”，不拿短周期冒充周额度，也不伪造 `0%`。

### 4.3 重置时间

重置时间按设备本地时区显示。设备时区或系统时间不正确时，显示时间也可能不正确。

### 4.4 Credits 与可用重置次数

部分账号可能返回 workspace credits 或 rate-limit reset credits。它们是可选数据：

- 未显示不一定代表数量为零，也可能是服务端没有提供该字段。
- 可用重置次数以服务端返回的总数为准。
- 服务端返回的到期时间会去重、按时间排序并使用设备本地时区显示。
- QuotaGlance 只展示信息，不会自动或手动消耗重置次数。

## 5. 三种窗口状态

### 5.1 浮球模式（orb）

浮球是默认的快速查看状态。当前可见球体为 128 × 128 像素，位于 136 × 136 像素透明窗口中，四周留出少量边缘光效空间。

浮球主要显示：

- 球体顶部的“周额度”标题和动态剩余比例。
- 水面下方的重置日期与状态文案；浏览器预览中的 `96%` 和 `7 月 20 日重置` 仅用于视觉验收，桌面应用始终读取真实快照。
- 横穿球体中部并随剩余比例在安全区间内升降的动态水面；数字与圆环负责精确比例，水面保留重置日期和状态的可读空间，并像装有液体的球体一样左右倾斜、轻微回弹。颜色跟随当前主题，系统开启“减少动态效果”后保持静止。
- 完整圆形的多层玻璃高光、球面明暗、内沿光效和圆形额度环，不再使用圆角矩形外观。
- 正常、提醒、危险、未知或旧数据状态符号与文字；不同主题和状态分别使用经过校准的球壳区、液体区文字颜色。
- 刷新中的轻量提示。

默认选择规则：

1. 优先选择用户固定额度桶中的周额度，不依赖该窗口位于 `primary` 还是 `secondary`。
2. 固定额度桶没有周额度时，选择其他桶中最先返回的周额度。
3. 所有桶都没有周额度时显示不可用状态。

当前交互：

- 双击浮球或按 Enter/Space：展开为卡片。
- 右键浮球，或聚焦后按菜单键 / Shift+F10：打开只含“设置”和“退出”的系统原生菜单。
- 选择“设置”：先展开卡片，再打开设置面板。
- 选择“退出”：真正结束 QuotaGlance，并回收 App Server 子进程。
- 按住浮球左键并移动超过 5px：拖动原生窗口；轻点不会误触，拖动后仍可正常双击展开。
- 通过卡片或托盘：切换置顶与鼠标穿透。

刷新、置顶和解除鼠标穿透不放进浮球右键菜单。它们仍可从展开卡片或系统托盘使用；这样右键菜单保持简洁，同时鼠标穿透后的托盘恢复入口不会丢失。

### 5.2 展开卡片（card）

展开卡片用于查看完整信息，计划宽高约为 320 像素，并根据实际内容调整。

信息从上到下包括：

1. 产品名、当前套餐和刷新状态。
2. 周额度剩余比例与进度。
3. 周额度重置时间。
4. 服务端实际返回的短周期/5 小时额度与重置时间，以整行详情显示。
5. 可用重置机会总数及可展开的到期时间，以独立整行详情显示。
6. 最后成功更新时间和数据新鲜度。
7. 刷新、置顶、收起和设置入口；操作结果使用短暂提示，不覆盖数据新鲜度。

卡片中缺少某一窗口或可选字段时，其余内容仍应正常显示。

### 5.3 隐藏模式（hidden）

隐藏模式不显示桌面窗口，只保留 Windows 系统托盘或 macOS 菜单栏图标。

- 关闭浮球或卡片时，默认进入隐藏模式，不直接退出应用。
- 隐藏期间仍会以更低频率进行安全重同步。
- 从托盘选择“显示”可恢复窗口。
- 只有选择“退出 QuotaGlance”才会结束应用和 App Server 子进程。

## 6. 窗口控制

### 6.1 置顶

置顶使浮球或卡片保持在普通窗口之上。置顶与鼠标穿透是两个独立开关：

- 置顶开启，不代表无法点击。
- 鼠标穿透开启，不代表一定置顶。

### 6.2 鼠标穿透

开启鼠标穿透后，鼠标事件会传递给浮球下方的应用。此时不能直接点击或拖动浮球。

恢复方法：

1. 打开系统托盘或 macOS 菜单栏中的 QuotaGlance 菜单。
2. 选择“解除鼠标穿透”或“解除锁定”。
3. 如窗口仍不可见，再选择“显示窗口”。

托盘中的解锁入口是强制保留的安全出口。

### 6.3 多显示器与缩放

QuotaGlance 计划记住窗口位置、尺寸和所在显示器。外接显示器移除或分辨率变化后，应用应把窗口调整回当前可见工作区。

如果自动恢复失败，可从托盘选择“显示窗口”或“重置窗口位置”。最终菜单名称以正式版为准。

## 7. 系统托盘与菜单栏

1.0 计划提供以下菜单项：

| 菜单项 | 作用 |
|---|---|
| 显示/隐藏 | 在当前窗口状态与隐藏状态之间切换 |
| 刷新额度 | 请求完整读取当前账号和额度 |
| 始终置顶 | 开启或关闭窗口置顶 |
| 解除鼠标穿透 | 在窗口无法点击时恢复控制 |
| 登录时启动 | 设置登录系统后是否自动运行；保存失败时不保留半完成状态 |
| 检查更新 | 手动检查签名更新 |
| 设置 | 打开显示模式、主题和窗口偏好 |
| 退出 QuotaGlance | 真正结束应用 |

macOS 使用菜单栏图标，Windows 使用系统托盘图标，文字和布局可能遵循平台习惯略有不同，但功能口径应保持一致。

## 8. 刷新与数据新鲜度

### 8.1 自动更新

额度变化主要由 App Server 通知触发。通知可能只包含部分信息，因此 QuotaGlance 不直接把通知载荷当作完整快照，而是在合并短时间内的重复通知后重新读取全部额度。

为了防止漏掉通知，当前实现还会执行安全重同步：

| 场景 | 计划行为 |
|---|---|
| 应用启动 | 完成初始化后立即读取 |
| 额度变化通知 | 合并通知后完整重读 |
| 账号状态变化 | 重新读取账号和额度 |
| 窗口可见 | 每 5 分钟安全重同步 |
| 仅托盘驻留 | 每 10 分钟安全重同步 |
| 睡眠或网络恢复 | 重新连接；快照超过 60 秒时重读 |

### 8.2 手动刷新

可从展开卡片或托盘选择“刷新额度”。

- 一次刷新会完整读取当前额度。
- 30 秒内重复点击会被合并，不会产生多次相同请求。
- 刷新中会显示进度状态。
- 刷新失败且已有成功数据时，旧值仍会保留。

### 8.3 如何判断数据是否新鲜

展开卡片计划显示：

- 最近一次尝试刷新的状态。
- 最后一次成功更新时间。
- 当前数据是否为旧数据。
- 如果可计算，显示下一次自动重试时间。

判断任务是否继续前，应先留意“最后成功更新时间”，尤其是在断网、睡眠恢复或服务异常之后。

## 9. 状态和错误提示

| 界面状态 | 含义 | 建议操作 |
|---|---|---|
| 正在读取 | 首次结果尚未返回 | 等待片刻；超过 15 秒后查看具体提示 |
| 正常 | 最近一次读取成功 | 无需操作 |
| 旧数据 | 最近刷新失败，显示的是最后成功值 | 检查网络并手动刷新；不要把旧值当成当前精确值 |
| 未登录 | 当前需要 OpenAI 认证，但没有可用登录 | 在 Codex 官方客户端或 CLI 登录后再刷新 |
| API Key 模式 | 当前使用标准 API 计费 | 订阅额度窗口不适用；到官方 API 控制台查看 API 用量 |
| 当前认证模式不适用 | 例如 Bedrock 或尚未支持的新认证模式 | 核对 Codex 当前认证方式；不要把它理解为额度为零 |
| 已触顶 | 服务端报告某个额度窗口已达到限制 | 查看对应窗口和重置时间 |
| 离线 | 网络、代理或上游连接不可达 | 恢复网络，检查代理，然后手动刷新 |
| 数据源繁忙 | App Server 或服务返回可重试错误 | 等待自动退避，稍后再刷新 |
| 服务暂不可用 | Codex 服务当前不可用 | 保留旧值，稍后重试 |
| App Server 缺失 | 随包组件不完整或外部 CLI 路径无效 | 修复安装或重新选择官方 CLI |
| App Server 过旧 | 当前版本不支持所需协议 | 升级 QuotaGlance 或所选 CLI |
| 系统拒绝执行 | 文件权限、安全软件或安装来源阻止运行 | 核对签名、安装来源和安全软件记录 |
| 当前版本暂不兼容 | 返回结构发生变化，无法安全解析 | 升级 QuotaGlance；不要依赖屏幕上的旧值 |

更具体的处理步骤见 [故障排查指南](./troubleshooting.md)。

## 10. 设置

当前设置使用覆盖卡片主体的完整玻璃面板，提供“跟随系统、极光、石墨、纸白、日落珊瑚、蜂蜜琥珀、玫瑰铜夜”七个主题选项，并以单行圆形色样排列。切换后卡片、浮球球壳、水面、水线、气泡、边框和控件会同步变化并自动保存。重新启动应用后会恢复上次选择；选择“跟随系统”时会根据操作系统外观应用明暗令牌。

设置面板支持完整键盘操作：Tab 与 Shift+Tab 只在面板控件间循环，方向键、Home 和 End 可切换主题，Escape 关闭后焦点会回到设置按钮。当前显示模式、置顶、鼠标穿透和登录时启动都使用一致的开关与保存状态。

当前已经保存窗口模式、置顶、鼠标穿透、主题和登录时启动。1.0 还计划完善以下本地偏好：

- 窗口位置、尺寸和窗口模式。
- 置顶和鼠标穿透状态。
- 简体中文或英文。
- 七套外观主题（已实现）。
- 登录时启动（已实现，仍待 macOS/Linux 实机验收）。
- 用户选定的主额度桶。
- 更新检查时间和更新偏好。

偏好使用本地 JSON 文件保存。若配置损坏，应用会从有效备份恢复，或以默认设置启动。

## 11. 隐私与安全

### 11.1 QuotaGlance 计划读取什么

- App Server 返回的认证类型和可选套餐类型。
- 额度桶、窗口、已用比例、重置时间和服务端触顶分类。
- 可选的 credits 和可用重置次数摘要。
- 应用自己的窗口、主题、语言和启动偏好。

### 11.2 QuotaGlance 不应读取或保存什么

- Token、API Key、刷新 Token 或 Authorization Header。
- auth.json 内容。
- 系统 keyring 内容。
- 账号 ID、电子邮箱和组织标识。
- 提示词、对话内容、代码、项目文件或 Codex 会话历史。
- 默认的额度历史和行为画像。

### 11.3 网络行为

- React 界面不直接请求 ChatGPT 或 Codex 服务。
- 正式主路线由本地 Codex App Server 访问官方服务。
- 不实现直接读取 `auth.json` 的备用 Provider，也不调用非公开 `wham` HTTP 接口。
- QuotaGlance 与 App Server 默认通过本机 stdio JSONL 通信，不开放本地网络端口。
- 1.0 默认不启用遥测。

### 11.4 诊断信息

发生故障时，不要发送：

- auth.json。
- Token 或 API Key。
- 完整原始协议消息。
- 含用户名的绝对路径。
- 账号邮箱、账号 ID 或组织 ID。

只提供错误码、应用版本、操作系统版本、App Server 版本、问题发生时间和已脱敏日志。具体清单见故障排查指南。

## 12. 更新与退出

### 12.1 更新

QuotaGlance 计划在启动后延迟检查一次更新，此后每天检查一次。更新应满足：

- 更新包必须通过签名验证。
- 默认由用户确认下载和重启。
- 不进行静默强制更新。
- 更新失败不影响当前额度查询。

正式版本发布前，不会有可用的应用内更新服务。

### 12.2 退出

关闭窗口默认只隐藏。要真正退出：

1. 打开系统托盘或 macOS 菜单栏菜单。
2. 选择“退出 QuotaGlance”。
3. 应用退出时会结束由自身启动的 App Server 子进程。

## 13. 卸载与本地数据

正式版卸载方式预计遵循系统标准：

- Windows：从“设置 → 应用”卸载。
- macOS：退出应用后，从“应用程序”移除 QuotaGlance。

1.0 默认不保存额度历史。卸载后可能仍保留偏好配置，是否提供“同时删除本地设置”选项将在实现阶段确认。任何卸载说明都不得要求用户删除 Codex 的 auth.json 或系统凭据，因为这些凭据属于 Codex，而不是 QuotaGlance。

## 14. 获取帮助

提交问题前，请先阅读 [故障排查指南](./troubleshooting.md)，并准备以下脱敏信息：

- QuotaGlance 版本。
- 操作系统版本与 CPU 架构。
- App Server 版本及来源类型（随包或外部 CLI）。
- 问题发生时间与界面错误码。
- 是否使用代理、VPN 或企业网络。
- 能稳定复现问题的操作步骤。

请勿附加凭据文件或敏感原始响应。
