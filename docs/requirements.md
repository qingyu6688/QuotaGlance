## 2. 产品定义

### 2.1 产品定位

QuotaGlance（额度一览）是一款面向 Codex 高频用户的 Windows 与 macOS 本地桌面工具。它计划通过浮球、展开卡片和系统托盘展示当前 Codex 额度桶、剩余比例、重置时间和数据状态，帮助用户在不中断编码工作的情况下安排任务节奏。

QuotaGlance 是独立第三方工具，与 OpenAI 无隶属、授权或背书关系。

### 2.2 产品目标

- 让用户在几秒内判断当前额度是否适合继续执行高消耗任务。
- 使用 Codex App Server 的公开本地协议读取账号状态和 ChatGPT 额度信息。
- 将服务端返回的动态额度桶转换为一致、易读且不失真的界面，覆盖套餐、可选短周期/5 小时窗口、周额度、重置机会与到期时间。
- 在断网、登录失效、App Server 异常或协议变化时给出可区分的状态。
- 正常路径不读取凭据文件、不访问系统凭据库、不持有 Token。
- 在 Windows 与 macOS 上提供安静、可恢复、低资源占用的常驻体验。

### 2.3 非目标

1. 不自建 Codex 登录页、认证中转服务或云端账户体系。
2. 不上传、同步、备份或展示 Codex Token。
3. 不调用登录、退出、购买、兑换或其他账号写操作。
4. 不调用 account/rateLimitResetCredit/consume，也不提供兑换重置次数的界面。
5. 不根据本地 token 数量或客户端活动推算服务端额度。
6. 不把 API Key 按量计费信息与 ChatGPT 订阅额度显示在同一进度条中。
7. 1.0 不保存额度历史，不提供趋势或预计耗尽时间。
8. 1.0 不提供云同步、团队管理后台、移动端或多 Provider。
9. 1.0 不使用 MySQL、Redis、消息队列或本地 REST 服务。
10. 不扫描私有缓存、临时目录或任意应用包；预览版仅允许固定外部安装位置、`PATH` 和常见 CLI 目录，正式版仍需验证来源与签名。

## 4. 典型场景

### SC-001 开始高消耗任务

用户准备进行大型重构，通过浮球或展开卡片确认最短额度窗口的剩余比例和重置时间，再决定立即开始或推迟任务。

### SC-002 深度工作中快速查看

用户不切换 IDE，通过置顶浮球查看最关键额度。浮球的信息不能只依赖颜色表达。

### SC-003 接近或达到额度上限

额度进入提醒或危险区间时，界面明确显示剩余比例、额度桶名称和重置时间；服务端报告触顶时，界面显示服务端分类，不用本地猜测替代。

### SC-004 额度结构因账号而异

某些账号只有一个窗口，另一些账号返回多个额度桶或多个窗口。界面按实际返回内容展示，缺少周窗口不视为失败。

### SC-005 登录方式不适用订阅额度

用户当前使用 API Key、Amazon Bedrock 或其他非 ChatGPT 订阅模式。界面说明当前认证模式与订阅额度视图不匹配，不显示伪造的 0%。

### SC-006 网络或数据源短暂失败

刷新失败但存在历史成功快照时，界面保留最后成功值并标记为旧数据，同时显示最后成功时间和下一步操作。

### SC-007 睡眠、换网和多屏变化

设备从睡眠恢复、网络切换或外接显示器被移除后，应用重新连接数据源，并确保窗口回到当前可见工作区。

### SC-008 鼠标穿透后找回控制权

用户启用鼠标穿透后无法点击浮球，仍可从托盘菜单解除锁定、显示窗口或退出应用。

## 5. 版本优先级

### 5.1 P0：1.0.0 必须完成

- App Server 启动、初始化握手、版本检查、异常退出检测和重启退避。
- 通过 account/read 读取账号状态，通过 account/rateLimits/read 读取额度。
- 动态额度桶、窗口、剩余比例、重置时间、可选 credits 和重置次数展示。
- 更新通知、手动刷新、安全重同步、缓存和失败保留最后成功值。
- 浮球、展开卡片、隐藏三种窗口状态。
- 托盘、拖动、置顶、鼠标穿透、窗口位置恢复、单实例和开机启动。
- 简体中文、英文，以及跟随系统、极光、石墨、纸白、日落珊瑚、蜂蜜琥珀、玫瑰铜夜主题。
- Windows 11 x64 和 macOS 13+ Intel/Apple Silicon 的安装与实机验收。
- 签名发布、macOS 公证、签名更新检查和完整隐私边界。

### 5.2 P1：1.1.x 建议完成

- Windows ARM64 构建与实机验证。
- 用户自定义低额度提醒阈值和安静时段。
- 托盘图标展示用户选择的两个额度桶或窗口。
- 用户主动导出的默认脱敏诊断包。
- Stable/Beta 更新通道。
- 企业代理和受管网络的增强诊断。

### 5.3 P2：后续独立评估

- 用户主动开启的 7 天或 30 天本地额度历史。
- 与额度窗口明确分离的本地活动摘要。
- 明确标注为预测的消耗速率和预计耗尽时间。
- 多账号手动切换。
- Claude、Gemini 等多 Provider。
- 需要独立权限与合规设计的团队级汇总。

| NFR-004 | 资源 | 应用与 App Server 合计空闲内存目标不高于 150 MiB | 双平台实测，记录设备与版本 | 规划 |
| NFR-005 | 可靠性 | 畸形消息、缺失字段、未知通知和进程退出不得导致应用崩溃 | 契约、模糊和恢复测试 | 规划 |
| NFR-006 | 一致性 | 同类 App Server 读取同一时间最多一个 | 并发自动化测试 | 规划 |
| NFR-007 | 可恢复性 | 断网、换网、睡眠和外接屏移除后可自动恢复或给出明确状态 | 双平台实机矩阵 | 规划 |
| NFR-008 | 隐私 | 正常路径不读取 auth.json、不访问 keyring、不持有或记录 Token | 代码审查、权限审查和运行时观察 | 规划 |
| NFR-009 | 安全 | WebView 不具备通用网络、文件系统或 Shell 权限；App Server 使用本地 stdio JSONL | Tauri Capability 与进程检查 | 规划 |
| NFR-010 | 安全 | 单条协议消息有大小上限、请求有超时，日志默认脱敏 | 边界测试和日志抽查 | 规划 |
| NFR-011 | 可访问性 | 键盘可操作、焦点顺序明确、状态不只依赖颜色，文本对比度至少达到 WCAG AA | 自动扫描与人工测试 | 规划 |
| NFR-012 | 国际化 | 中文和英文下无截断、错位或未翻译的核心文案 | 两种语言截图与 UI 测试 | 规划 |
| NFR-013 | 平台 | Windows 11 x64、macOS 13+ Intel/Apple Silicon 达到同一功能口径 | 发布实机矩阵 | 规划 |
| NFR-014 | 发布 | Windows 安装包签名；macOS 应用签名、公证并完成 stapling | 系统签名工具验证 | 规划 |
| NFR-015 | 更新 | 更新包必须通过独立更新签名验证，失败可安全回退 | 升级、篡改和失败恢复测试 | 规划 |
| NFR-016 | 数据 | 1.0 额度快照仅保存在内存，不形成默认历史记录 | 存储检查与卸载检查 | 规划 |
| NFR-017 | 可维护性 | TypeScript 严格模式；Rust 可恢复错误不得依赖 panic；协议解析与 UI 解耦 | lint、clippy、测试和代码审查 | 规划 |
| NFR-018 | 合规 | 关于页和发行文档包含非官方声明及随包组件许可信息 | 发布清单人工复核 | 规划 |

## 8. 状态模型

| 状态 | 判定 | 用户可见行为 |
|---|---|---|
| loading | 首次读取尚未完成 | 显示加载状态，不显示假百分比 |
| ok | 最近一次读取成功 | 显示额度、重置时间和更新时间 |
| stale | 读取失败但存在历史成功值 | 保留旧值，标记“数据可能已过期”并显示最后成功时间 |
| signedOut | 需要 OpenAI 认证且当前无可用账号 | 引导用户在 Codex 官方客户端或 CLI 完成登录 |
| apiKeyMode | 当前为 API Key 模式 | 说明使用标准 API 计费，不适用 ChatGPT 订阅额度窗口 |
| unsupportedAuth | 当前认证模式不属于 1.0 可展示范围 | 显示认证模式不适用或暂不支持，不猜测额度 |
| offline | 本机网络、代理或上游连接不可达 | 保留旧值并低频重试 |
| quotaReached | 服务端返回已触顶分类 | 显示触顶状态和可用重置时间 |
| sourceBusy | App Server 或上游返回可重试错误 | 显示稍后重试，按退避策略恢复 |
| serviceUnavailable | Codex 服务暂时不可用 | 保留旧值并显示服务状态 |
| appServerMissing | 找不到可执行的数据源 | 显示修复或重新安装指引 |
| appServerOutdated | 版本不满足协议要求 | 提示升级 QuotaGlance 或所选外部 CLI |
| appServerDenied | 系统拒绝执行 App Server | 说明权限或安装来源问题，不尝试读取 Token 绕过 |
| incompatible | 返回结构无法安全解析 | 提示升级应用，禁止显示 0% 替代 |

## 9. 数据与隐私边界

### 9.1 允许处理

- 由 App Server 返回的认证类型、可选套餐类型。
- 动态额度桶、窗口、已用比例、重置时间和服务端触顶分类。
- 可选的 workspace credits 和 rate-limit reset credits 摘要。
- 窗口位置、窗口模式、主题、语言、置顶、穿透和更新偏好。
- 不含敏感内容的错误码、版本号和时间戳。

### 9.2 不应进入前端、日志或配置

- Access Token、API Key、刷新 Token 或 Authorization Header。
- ChatGPT 账号 ID、电子邮箱和组织标识。
- auth.json 内容及其绝对路径。
- 系统 keyring 内容。
- App Server 原始协议响应。
- 提示词、对话、代码、项目路径或 Codex 会话历史。

### 9.3 凭据存储兼容

Codex 官方凭据存储配置包括 file、keyring 和 auto。正常路径由 App Server 负责读取和刷新凭据；QuotaGlance Core 不直接处理这三种存储。file 模式可能将凭据保存在 CODEX_HOME 下的 auth.json，该文件应被视为密码文件，用户不得将其作为诊断附件发送。

### 10.2 P0 需求追踪规则

- FR-001 至 FR-041、NFR-001 至 NFR-018 默认均为 1.0 验收范围。
- P0 需求未全部达到“已验收”时，不得将版本标记为 1.0.0 正式发布。
- POC 结果若推翻 App Server 可行性，应先修订本文档和架构；不得读取 `auth.json`/Token，也不得调用非公开 `wham` 接口作为降级路径。

## 11. 约束与待决事项

1. M0 必须验证 App Server sidecar 在 Windows x64、macOS Intel 和 Apple Silicon 上的合法分发、签名与启动。
2. `0.1.4` 外部候选允许列表仅包含 Windows 受管运行时、`PATH`、macOS `/Applications` 或 `~/Applications` 下的 `ChatGPT.app`/`Codex.app` 固定资源路径，以及常见 CLI 目录；macOS 优先统一版 ChatGPT，再兼容旧版 Codex，且不得扩展为任意扫描。候选尚未完成 OpenAI 签名身份验证。
3. App Server 认证模式和返回字段可能继续扩展，解析器必须对未知值安全降级。
4. rateLimitsByLimitId 为首选多桶视图；rateLimits 是向后兼容单桶视图，二者的优先级不可颠倒。
5. account/rateLimits/updated 可能只携带部分单桶信息，1.0 应始终在通知合并后完整重读。
6. Windows 10 22H2 只对仍获 ESU 安全更新的设备尽力支持，不作为阻塞发布的正式基线。
7. Apple Developer 账号、Windows 签名方案和 CI Secrets 需要在功能完成前准备。
