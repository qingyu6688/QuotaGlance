# QuotaGlance 故障排查指南

> 文档版本：0.1.0 开发预览版 / 1.0 规划版  
> 对应产品版本：0.1.0（源码开发基线），1.0.0（目标）  
> 更新日期：2026-07-12  
> 维护邮箱：maorongkang@gmail.com

## 1. 文档说明

本文档按“症状 → 可能原因 → 处理方法 → 需收集的脱敏信息”组织，用于 QuotaGlance 1.0 的开发验收和正式发布后的用户排障。

**当前没有正式发布版；只提供本地 Windows x64 未签名 Release 预览安装包。** 系统显示未知发布者或 SmartScreen 提示属于预期限制，不应通过关闭系统安全机制绕过。开发预览问题仍应在项目根目录按开发命令排查。

开发预览已经包含前端状态界面、只读类型化协议、刷新恢复和基础偏好持久化，但真实账号、可合法分发的 production sidecar、完整设置、更新器、macOS 和签名安装器尚未完成。遇到这些缺口时，不要通过读取凭据文件或关闭系统安全机制绕过。

## 2. 排障原则

开始排查前，先遵守以下规则：

1. 不要发送或粘贴 auth.json。
2. 不要发送 Token、API Key、刷新 Token 或 Authorization Header。
3. 不要发送完整 App Server 原始请求或响应。
4. 不要截图包含邮箱、账号 ID、组织 ID 或个人文件路径的界面。
5. 不要通过关闭 Gatekeeper、修改 WindowsApps 权限或全局禁用安全软件来绕过故障。
6. 先记录问题发生时间和错误码，再尝试重启或重新安装，避免丢失线索。
7. 额度未知、旧数据和额度为零含义不同，不要互相替代。

安全提交的信息通常包括：

- QuotaGlance 版本号。
- 操作系统版本和 CPU 架构。
- App Server 版本号及来源类型（随包或外部 CLI）。
- QuotaGlance 展示的脱敏错误码。
- 问题发生时间和最后成功更新时间。
- 网络类型（家庭、公司、代理、VPN），不含代理密码。
- 可重复的操作步骤。

## 3. 建议排查顺序

遇到额度读取问题时，按以下顺序判断：

1. 先确认当前运行的是源码开发预览还是未来的正式发布版；当前没有可验证签名的正式安装包。
2. 查看卡片中的状态名称、错误码和最后成功更新时间。
3. 确认 Codex 官方客户端或 CLI 当前能否识别登录状态。
4. 确认当前认证方式是否适用于 ChatGPT 订阅额度。
5. 检查网络、代理、VPN 和企业安全策略。
6. 从托盘执行一次手动刷新，并等待 30 秒冷却结束。
7. 如提示 App Server 缺失、过旧或拒绝执行，先修复数据源，不要通过读取凭据文件绕过。
8. 仍无法恢复时，按本文档末尾模板收集脱敏信息。

## 4. 安装与启动

| 症状 | 可能原因 | 处理方法 | 需收集的脱敏信息 |
|---|---|---|---|
| 找不到可下载的正式版本 | 项目仍在开发；发布渠道尚未建立 | 查看项目发布页和更新日志。不要从第三方网盘下载未签名构建 | 当前访问的发布渠道、查询日期 |
| 源码开发预览无法启动 | Node.js、Rust、Tauri 系统依赖缺失，或尚未安装项目依赖 | 在项目根目录执行 `npm ci` 和 `npm run check`；前端预览使用 `npm run dev`，桌面调试使用 `npm run tauri -- dev` | Node.js、npm、Rust 版本，以及不含个人路径和凭据的错误摘要 |
| Windows 显示“未知发布者” | 安装包未签名、签名失效或下载来源不正确 | 取消安装，核对正式发布页的版本与签名信息；重新下载签名包 | 安装包版本、下载来源域名、Windows 版本、签名验证结果；不要发送整个安装包到公开工单 |
| Windows 安装后无法启动 | 安装不完整、WebView2 异常、安全软件拦截或 sidecar 被隔离 | 从系统设置修复或重装；检查安全软件隔离记录；只对官方签名文件恢复信任 | 应用版本、Windows 版本、WebView2 版本、脱敏拦截规则名称、错误码 |
| macOS 提示无法验证开发者 | 安装包不是正式签名公证版本，或下载损坏 | 删除该副本，从正式渠道重新下载并核对签名；不要关闭 Gatekeeper | macOS 版本、CPU 架构、应用版本、Gatekeeper 提示原文（去除用户名和路径） |
| macOS 提示应用已损坏 | 下载不完整、公证票据异常、架构不匹配或文件被修改 | 重新下载适配架构的 DMG；核对校验值；仍失败时报告问题 | macOS 版本、Intel/Apple Silicon、DMG 版本、校验结果 |
| 启动后没有任何窗口 | 应用以隐藏模式启动、窗口在屏幕外、单实例已运行或启动失败 | 查看托盘/菜单栏；选择“显示窗口”或“重置窗口位置”；确认任务管理器或活动监视器中是否已有实例 | 应用版本、显示器数量和缩放、托盘是否存在、错误码 |
| 再次启动没有新窗口 | 单实例机制将操作转交给已运行实例 | 从托盘选择“显示”；如无响应，正常退出已有实例后重启 | 操作系统、应用版本、现有进程数量、托盘状态 |

## 5. App Server 数据源

| 症状 | 可能原因 | 处理方法 | 需收集的脱敏信息 |
|---|---|---|---|
| 显示“App Server 缺失” | 随包组件缺失、被安全软件隔离，或开发模式选择的外部 CLI 路径无效 | 正式版执行修复安装；开发模式重新选择可信的 Codex CLI；不要手动复制来路不明的可执行文件 | QuotaGlance 版本、App Server 来源类型、CPU 架构、脱敏错误码、安全软件事件名称 |
| 显示“App Server 过旧” | 外部 CLI 或随包组件版本不支持所需协议 | 正式版升级 QuotaGlance；开发模式升级外部 Codex CLI 后重新选择 | QuotaGlance 版本、Codex/App Server 版本、来源类型 |
| 显示“系统拒绝执行” | 文件权限、企业执行策略、签名策略或安全软件阻止子进程 | 核对安装来源和签名；联系设备管理员放行正式签名版本；不要修改 WindowsApps ACL 或全局关闭安全策略 | 操作系统、错误码、签名状态、策略或安全软件的脱敏规则名 |
| Windows 中能看到 Codex 桌面应用文件，但启动 App Server 被拒绝 | WindowsApps 内部二进制受系统保护，不能作为外部工具依赖 | 使用 QuotaGlance 随包 sidecar；开发期使用单独安装且可执行的 Codex CLI。不要接管 WindowsApps 文件权限 | Windows 版本、QuotaGlance 版本、数据源来源类型；路径只保留末级文件名 |
| App Server 反复重启 | 组件崩溃、架构不匹配、初始化失败或安全软件终止进程 | 等待退避停止高频重启；修复安装；核对 CPU 架构；升级到匹配版本 | 重启次数、首次失败时间、QuotaGlance/App Server 版本、错误码 |
| 显示“初始化失败” | initialize 参数不兼容、连接中断或 App Server 已收到重复初始化 | 重启 QuotaGlance；升级应用；开发构建需检查每个连接只初始化一次，且 initialized 在成功响应之后发送 | 两端版本、错误码、初始化阶段名称；不要附加原始协议全文 |
| 显示“当前版本暂不兼容” | App Server 返回结构发生变化，解析器无法安全处理 | 升级 QuotaGlance；保留最后成功值但不要继续把它当作实时数据 | 应用与 App Server 版本、字段级脱敏解析错误、发生时间 |
| 外部 CLI 在终端可用，但 QuotaGlance 找不到 | GUI 环境的 PATH 与终端不同，或路径未被明确授权 | 在设置中选择 CLI 的真实安装路径；正式版优先使用随包组件 | 操作系统、Shell 类型、CLI 版本、仅保留磁盘和文件名的脱敏路径 |

## 6. 登录与认证

| 症状 | 可能原因 | 处理方法 | 需收集的脱敏信息 |
|---|---|---|---|
| 显示“未登录” | Codex 登录已退出、会话失效，或受管策略清除了凭据 | 在 Codex 官方客户端或 CLI 完成登录；确认官方客户端可识别登录后回到 QuotaGlance 刷新 | 认证类型（不要提供账号）、Codex 版本、是否受企业策略管理、错误码 |
| 已在浏览器登录 ChatGPT，但仍显示未登录 | 浏览器登录与本地 Codex 凭据不是同一状态 | 使用 Codex 官方客户端或 CLI 登录；仅登录网页不足以证明 App Server 可使用本地会话 | 使用的 Codex 客户端类型、认证状态命令的结论；不要粘贴令牌输出 |
| 显示“API Key 模式”且没有订阅额度 | API Key 使用标准 API 计费，不等同于 ChatGPT 订阅额度 | 到官方 API 控制台查看 API 用量；如需订阅额度，按组织政策在 Codex 官方客户端切换到适用的 ChatGPT 登录 | 只提供认证模式 apikey，不提供 Key 内容；套餐预期和应用版本 |
| 显示“当前认证模式不适用” | 当前为 Bedrock、agent identity、personal access token 或新出现的认证方式 | 核对该模式是否由 Codex 服务提供 ChatGPT 额度；不要反复登录或把它当成额度为零 | authMode 名称、App Server 版本、错误码；不提供账号或凭据 |
| 登录后状态没有更新 | account/updated 通知未收到、App Server 连接中断或缓存尚未失效 | 从托盘手动刷新；如仍无变化，正常退出并重启 QuotaGlance | 登录完成时间、刷新时间、应用与 App Server 版本、通知计数等脱敏诊断值 |
| file 模式可登录，但 keyring 或 auto 模式失败 | 系统凭据库不可用、权限策略限制或 App Server 兼容问题 | 在 Codex 官方客户端验证相同存储模式；升级 Codex；联系管理员检查系统凭据库策略。QuotaGlance 不应直接读取 keyring 代替 App Server | 操作系统、cli_auth_credentials_store 取值、App Server 版本、错误码 |
| 被要求提供 auth.json 才能排查 | 排障流程不符合项目隐私规范 | 拒绝提供。只提交认证模式、错误码和版本信息 | 不适用；不要收集该文件 |

## 7. 额度内容异常

| 症状 | 可能原因 | 处理方法 | 需收集的脱敏信息 |
|---|---|---|---|
| 一直显示“正在读取” | 初次请求超时、App Server 未完成初始化、网络不可达或上游无响应 | 等待至 15 秒明确错误；检查网络；手动刷新一次；再检查 App Server 状态 | 开始时间、超过时长、网络类型、两端版本、错误码 |
| 显示“旧数据” | 最近刷新失败，但应用保留了最后成功快照 | 查看最后成功时间；恢复网络或服务后刷新。做任务决策时不要把旧值当作当前精确值 | 最后成功时间、最近失败时间、错误码、网络状态 |
| 剩余比例看起来相反 | 已用比例到剩余比例的转换错误，或用户将两种口径混淆 | 核对界面标签是否为“剩余”；升级到修复版本；不要自行用原始响应对外报告 | 界面显示值、额度桶和窗口标签、应用版本；不发送原始响应 |
| 显示未知而不是 0% | 字段缺失、解析失败或数据源没有提供该值 | 这是安全降级，不应手动改成 0%；查看具体状态并升级或重试 | 额度桶标识、缺失字段名称、两端版本、错误码 |
| 只有一个窗口，没有周额度 | 当前账号或服务端未返回 secondary/周窗口 | 无需修复。QuotaGlance 只展示实际返回的数据 | 套餐类型（如可公开）、limitId、窗口时长；不提供账号标识 |
| 出现多个陌生额度桶 | App Server 按 limitId 返回多桶，部分 limitName 可能为空或新增加 | 以服务端标识展示；不要把未知桶猜成固定产品。若布局异常再报告 | limitId、可选 limitName、窗口数量、应用版本 |
| 额度已变化，但界面没有立即更新 | 通知延迟、通知丢失、处于隐藏低频模式或刷新冷却中 | 展开卡片查看最后更新时间；等待安全重同步或手动刷新 | 额度变化的大致时间、最后更新时间、窗口模式、两端版本 |
| 收到通知后只更新了一个桶 | 客户端错误地把部分通知当成完整快照 | 升级 QuotaGlance。正确行为应把通知视为失效信号并完整调用 account/rateLimits/read | 应用版本、受影响的 limitId、更新前后桶数量、错误码 |
| Credits 没有显示 | 服务端没有提供字段、当前账号不适用或字段解析失败 | 额度窗口可继续使用；查看是否有字段不可用提示。缺失不自动等于零 | 应用/App Server 版本、是否出现局部降级提示、套餐类型 |
| 可用重置次数与明细行数不同 | 服务端可能只返回部分明细；availableCount 才是权威总数 | 以界面总数为准，不用明细数量反推。QuotaGlance 不提供消耗入口 | availableCount、实际明细行数、应用版本；不发送明细中的不透明 ID |
| 重置时间明显错误 | 系统时区或时间不正确，或 Unix 秒解析错误 | 校正系统时间与时区；重启后刷新；仍错误时报告显示时间和时区 | 系统时区、显示时间、windowDurationMins、应用版本；避免精确位置数据 |
| 显示“已触顶”但仍有其他窗口 | 某一个额度桶或窗口触顶，不代表所有桶都触顶 | 展开卡片确认触顶的 limitId 和窗口；以服务端分类为准 | 触顶桶标识、窗口时长、重置时间、rateLimitReachedType |

## 8. 刷新、网络与代理

| 症状 | 可能原因 | 处理方法 | 需收集的脱敏信息 |
|---|---|---|---|
| 点击刷新没有立即再次请求 | 30 秒手动刷新冷却或已有同类请求在进行 | 等待冷却结束；观察刷新状态，不要连续点击 | 点击时间、冷却提示、在途请求状态、应用版本 |
| 显示“离线”但浏览器可以上网 | 代理只配置在浏览器、企业网络拦截子进程，或 Codex 服务不可达 | 在 Codex 官方客户端验证连接；检查系统代理和企业策略；不要提交代理密码 | 网络类型、是否使用系统代理/VPN、官方客户端是否可用、错误码 |
| 连接 VPN 后停止更新 | VPN 路由、证书检查或代理策略改变 | 暂停 VPN 验证是否恢复；按组织要求配置系统级网络；不要绕过企业安全策略 | VPN 类型、连接前后时间、操作系统、错误码 |
| 服务恢复后仍显示旧数据 | 自动重试尚在退避，或 App Server 连接尚未恢复 | 等待下一次重试；从托盘手动刷新；必要时正常重启应用 | nextRetryAt、最后成功时间、服务恢复时间、两端版本 |
| 数据源繁忙后刷新越来越慢 | 指数退避正在生效 | 等待界面显示的下一次重试时间；不要通过反复重启制造更多请求 | 失败次数、退避间隔、错误码、App Server 版本 |
| 从睡眠恢复后不更新 | 子进程连接断开、网络尚未恢复或快照新鲜度判断失败 | 等待网络连接稳定；显示窗口或手动刷新；仍失败时重启 | 睡眠和唤醒时间、唤醒后网络恢复时间、最后更新时间 |

## 9. 浮球、窗口和托盘

| 症状 | 可能原因 | 处理方法 | 需收集的脱敏信息 |
|---|---|---|---|
| 浮球点不到 | 已开启鼠标穿透 | 从托盘/菜单栏选择“解除鼠标穿透”或“解除锁定” | 操作系统、穿透状态、托盘是否可用 |
| 浮球拖不动 | 鼠标穿透开启、拖动区域失效或平台权限异常 | 先从托盘解除穿透；展开卡片后再尝试拖动；仍失败时重置窗口位置 | 窗口模式、缩放比例、操作系统、应用版本 |
| 窗口消失但托盘仍在 | 进入 hidden 模式，或窗口位于已移除显示器 | 选择“显示窗口”；再选择“重置窗口位置” | 显示器数量、拔插情况、缩放比例、窗口模式 |
| 托盘图标不见了 | 系统折叠图标、Explorer/Finder 状态异常或应用已退出 | 检查隐藏图标区域；确认进程；重启系统托盘进程或 QuotaGlance | 操作系统、应用进程是否存在、托盘区域截图（先遮盖其他敏感图标） |
| 关闭窗口后应用仍在 | 设计上关闭默认隐藏到托盘 | 从托盘选择“退出 QuotaGlance” | 不需要诊断；如退出失败再提供版本和进程状态 |
| 置顶失效 | 全屏应用、系统空间切换或平台窗口层级限制 | 切换一次置顶；退出全屏应用验证；记录发生场景 | 操作系统、目标应用是否全屏、窗口模式、应用版本 |
| 外接屏拔掉后窗口在屏幕外 | 工作区恢复逻辑未生效 | 从托盘执行“重置窗口位置”；重启后仍应回到主屏 | 拔插前后显示器布局、分辨率、缩放，不提供桌面截图中的私人内容 |
| 高 DPI 或 Retina 下模糊、过小 | 每显示器 DPI/Retina 适配异常 | 将窗口移回主屏验证；记录各显示器缩放；升级应用 | 操作系统、显示器分辨率与缩放、窗口所在屏幕 |

## 10. 设置与本地偏好

| 症状 | 可能原因 | 处理方法 | 需收集的脱敏信息 |
|---|---|---|---|
| 重启后窗口位置或主题未保存 | 偏好文件写入失败、目录只读或应用异常退出 | 正常退出后重试；检查用户配置目录是否可写；不要手动编辑未知字段 | 应用版本、设置项名称、错误码；路径仅保留通用目录名 |
| 启动后设置恢复默认值 | 配置损坏后应用执行安全回退 | 重新设置偏好；如反复发生，提交脱敏配置结构，不提交绝对路径 | 配置 schema 版本、损坏时间、错误码 |
| 开机启动未生效 | 系统启动项被禁用、企业策略限制或注册失败 | 在系统启动项设置中确认；重新切换一次；企业设备联系管理员 | 操作系统、启动项状态、策略提示、应用版本 |
| 语言切换后部分文字未变化 | 翻译资源缺失或窗口未刷新 | 重新打开卡片或重启；记录未翻译区域 | 语言、界面区域、应用版本、去敏截图 |
| 深色主题下内容看不清 | 主题变量、系统高对比度或 WebView 差异 | 切换跟随系统或浅色临时恢复；报告对比度问题 | 主题、系统高对比设置、操作系统、去敏截图 |

## 11. 更新与版本

| 症状 | 可能原因 | 处理方法 | 需收集的脱敏信息 |
|---|---|---|---|
| 检查更新失败 | 更新服务不可达、代理限制或系统时间错误 | 确认网络和系统时间；稍后重试。额度查询应继续工作 | 当前版本、错误码、网络类型、系统时间偏差 |
| 更新签名验证失败 | 下载损坏、更新元数据异常或文件被篡改 | 立即取消更新；从正式发布渠道手动下载安装包；不要强制跳过签名 | 当前与目标版本、签名错误码、下载来源 |
| 更新后 App Server 过旧 | 应用与随包组件版本组合不完整，或仍选择旧外部 CLI | 执行修复安装；切回随包数据源或升级外部 CLI | 应用版本、App Server 版本、来源类型 |
| 更新后设置丢失 | 配置迁移失败或旧配置损坏 | 重新设置偏好；不要从未知备份复制凭据文件 | 更新前后版本、配置 schema 版本、丢失的设置项 |
| 更新失败后应用无法启动 | 安装中断或回退失败 | 使用正式完整安装包执行修复；保留旧版本号和错误码 | 更新前后版本、操作系统、安装器错误码 |

## 12. 性能与稳定性

| 症状 | 可能原因 | 处理方法 | 需收集的脱敏信息 |
|---|---|---|---|
| 空闲 CPU 长时间偏高 | 刷新循环、子进程重启循环、动画或 WebView 异常 | 展开卡片查看刷新状态；正常重启；若持续则记录 10 分钟采样 | 应用和 App Server CPU、持续时间、窗口模式、两端版本 |
| 内存持续增长 | 快照、事件监听或日志未释放 | 重启临时恢复；记录增长速度和触发操作 | 起始/结束内存、观察时长、操作步骤、两端版本 |
| 应用偶发崩溃 | 畸形协议消息、平台窗口问题或组件异常 | 重启并记录崩溃前最后操作；升级到最新稳定版 | 崩溃时间、操作系统、两端版本、系统生成的脱敏崩溃摘要 |
| 隐藏后资源仍很高 | 隐藏调度未降频或 App Server 持续失败 | 从托盘查看状态；等待退避；正常退出后重启 | hidden 持续时间、刷新次数、CPU/内存、错误码 |

## 13. 何时需要提交问题

出现以下情况时，应提交问题而不是继续本地尝试：

- 正式签名安装包在支持的平台上无法启动。
- App Server 随包组件反复被判定缺失、过旧或架构不匹配。
- 刷新失败后把已知额度覆盖成 0%。
- API Key、Bedrock 或未知认证模式被错误显示为订阅额度耗尽。
- 收到额度通知后丢失其他额度桶。
- 错误信息、日志或诊断内容出现 Token、邮箱、账号 ID 或完整个人路径。
- 鼠标穿透后托盘没有解除锁定入口。
- 外接显示器移除后无法从托盘找回窗口。
- 更新签名失败但应用仍继续安装。
